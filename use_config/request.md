Тема: Архитектура CI/CD для Docker Compose с разделением конфигов и секретов через GitLab и Infisical.
Контекст: У меня есть несколько серверов, на которых приложения развернуты через Docker Compose. Конфигурация приложений хранится в файлах appsettings.json (или .env).
Стек:
CI/CD: GitLab.
Хранение секретов: Infisical.
Оркестрация: Docker Compose на нескольких хостах.
Задача: Помоги составить архитектурный план и примеры скриптов для решения следующих задач:
Хранение: Как лучше организовать хранение шаблонов конфигов (без секретов) в GitLab, чтобы их было удобно обновлять?
Интеграция: Напиши пример .gitlab-ci.yml, который авторизуется в Infisical, забирает секреты и прокидывает их в процесс деплоя на удаленные серверы через SSH.
Безопасность: Как реализовать доставку секретов так, чтобы на диске целевых серверов не оставалось открытых .env файлов? (Инъекция через infisical run в Docker Compose).
Масштабирование: Как эффективно обновлять эти настройки сразу на нескольких серверах (Target Servers) в рамках одного пайплайна?
Приложение: Если приложение требует appsettings.json, как лучше «смержить» открытый шаблон из Git и секреты из Infisical прямо в процессе деплоя?
Дай рекомендации по Best Practices для такой связки.
