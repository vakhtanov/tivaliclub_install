Пошаговая инструкция
Некоторые шаги для настройки Nginx в качестве обратного прокси:
Создать новый блок серверов для обратного прокси в каталоге /etc/nginx/sites-available/. Например, создать файл конфигурации reverse-proxy.conf.
Добавить конфигурацию обратного прокси. В файле нужно указать внутренний сервер, на который должны быть перенаправлены запросы, и директивы proxy_set_header для сохранения информации о клиенте.
Включить конфигурацию — создать символическую ссылку в директории sites-enabled.
Проверить конфигурацию Nginx, чтобы убедиться в отсутствии синтаксических ошибок: nginx -t.
Перезапустить Nginx, чтобы применить изменения: systemctl restart nginx.

Пример конфигурации для проксирования трафика на внутренний сервер, работающий на порту 3000:
server {  
    listen 80;  
    server_name yourdomain.com;  
    location / {  
        proxy_pass http://localhost:3000;  
        proxy_set_header Host $host;  
        proxy_set_header X-Real-IP $remote_addr;  
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;  
        proxy_set_header X-Forwarded-Proto $scheme;  
    }  
}  
``` [9](https://itshaman.ru/articles/5784/kak-nastroit-nginx-v-kachestve-obratnogo-proksi-na-ubuntu-2404)  

**Важно**: при TLS-терминации именно Reverse Proxy хранит SSL-сертификаты, и если сертификат истекает, весь защищённый трафик перестаёт работать. [10](https://blog.infra-tech.ru/obratny-proksi/)
Возможные ошибки
Некоторые ошибки, которые могут возникнуть при настройке Nginx в качестве обратного прокси, и способы их решения:
Потеря реального IP — не настроены заголовки X-Real-IP и X-Forwarded-For. Решение: добавлять в блок location директивы proxy_set_header.
Двойное проксирование — случайное использование proxy_pass внутри location / и вложенного location /api. Решение: убедиться, что proxy_pass используется только там, где это необходимо, и корректно настроены location для статики/динамики.
Ошибки 502 (Bad Gateway) — бэкенд-сервер недоступен или тайм-аут соединения. Решение: увеличить proxy_read_timeout и proxy_connect_timeout в блоке location, убедиться, что upstream-адрес и порт указаны верно.
Проблема с редиректами — бэкенд генерирует абсолютные ссылки с использованием HTTP вместо HTTPS. Решение: добавить заголовок proxy_set_header X-Forwarded-Proto $scheme; или настроить бэкенд на учёт этого заголовка.
